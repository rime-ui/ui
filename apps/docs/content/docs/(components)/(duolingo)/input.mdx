## Validation States

The input supports `error` and `success` variants. The `error` prop can take a string to display a validation message automatically.

## API Reference

### Props

| Prop         | Type                                     | Default     | Description                                |
| ------------ | ---------------------------------------- | ----------- | ------------------------------------------ |
| `label`      | `string`                                 | —           | The label displayed above the input.       |
| `variant`    | `'default' \| 'error' \| 'success'`      | `'default'` | The visual style variant of the input.     |
| `size`       | `'small' \| 'medium' \| 'large'`         | `'medium'`  | The size of the input field.               |
| `error`      | `string \| boolean`                      | —           | Shows error state. Displays text if string.|
| `helperText` | `string`                                 | —           | Helper text displayed below the input.     |
| `leftIcon`   | `ReactNode`                              | —           | Icon displayed on the left side.           |
| `rightIcon`  | `ReactNode`                              | —           | Icon displayed on the right side.          |
| `className`  | `string`                                 | —           | Additional classes for the wrapper.        |
| `dir`        | `'ltr' \| 'rtl'`                         | —           | Layout direction (useful for Arabic).      |

The `Input` component also accepts all standard `HTMLInputElement` attributes (like `type`, `placeholder`, `onChange`, `disabled`, etc.).
